# Philosophy Playground

> 用互動式應用理解抽象哲學概念

## 🎯 專案願景

這是一個開源專案，旨在將複雜的哲學、經濟學、心理學概念轉化為簡單易懂的互動式應用。

我們相信：**好的教育應該是有趣的、直觀的、可觸摸的。**

---

## 📦 已完成的應用

### 1. [囚徒困境](./prisoners-dilemma) | Prisoner's Dilemma

- **概念**：個人理性 vs 集體利益
- **互動方式**：與 AI 進行 5 輪對戰
- **學習目標**：理解為什麼「雙贏」如此困難

---

## 🚀 即將推出

- **公地悲劇**（Tragedy of the Commons）
- **沉沒成本謬誤**（Sunk Cost Fallacy）
- **確認偏誤**（Confirmation Bias）
- **帕累托法則**（80/20 Rule）

---

## 🎨 設計原則

1. **3 分鐘理解**：每個應用都應該在 3 分鐘內讓完全不懂的人理解核心概念
2. **視覺優先**：用圖像、動畫、互動取代長篇文字
3. **零門檻**：不需要任何背景知識，不需要安裝任何軟體
4. **開源免費**：所有程式碼公開，任何人都可以使用、修改、分享

---

## 💻 技術棧

- **純前端**：HTML + CSS + JavaScript
- **無依賴**：不使用任何外部函式庫（保持輕量）
- **響應式**：支援桌面與行動裝置

## 🤝 貢獻指南

歡迎任何形式的貢獻！

### 如何貢獻新應用

1. Fork 這個專案
2. 在 `philosophy-playground/` 下建立新資料夾（例如：`sunk-cost-fallacy/`）
3. 確保包含：
   - `index.html`（應用本體）
   - `style.css`（樣式）
   - `script.js`（互動邏輯）
   - `README.md`（概念說明）
4. 提交 Pull Request

### 設計規範

- 使用深色主題（背景 `#0a0e27`）
- 使用霓虹色調（藍 `#00d4ff`、紅 `#ff3366`、綠 `#00ff88`）
- 確保響應式設計（手機友善）
- 程式碼簡潔易讀（適合初學者學習）
